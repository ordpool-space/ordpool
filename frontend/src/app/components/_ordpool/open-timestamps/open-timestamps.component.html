OpenTimestamps: Anchor any file to Bitcoin

Anyone can claim they wrote a paper, took a photo, signed a contract, or held a leaked document on a given day. Proving it is harder. OpenTimestamps lets you prove a file existed at a specific Bitcoin block by hashing it, batching that hash with everyone else's into a Merkle tree, and committing the root in a single OP_RETURN. That root is now part of Bitcoin's permanent ledger. Anyone with the file, the .ots proof, and Bitcoin's block headers can verify "this existed by block N", forever, with no third party in the loop.

Ordpool tags every OTS transaction wherever it appears in the explorer, so you can see Bitcoin's notary at work in real time, and you can verify your own .ots receipt right here, against our independent open-source implementation. The whole process is free. Calendar operators sponsor the Bitcoin transaction fees and Merkle-batch every submission they receive into a single anchor. Huge thanks to the operators who keep this open public good alive.

Of course, the gold standard is to put the file itself on-chain as an inscription. But sometimes revealing the file isn't what you need: OTS commits only the hash, so the contents stay private and only you can ever prove what was timestamped. For everything you don't want to inscribe, OpenTimestamps is the next best thing: a tiny, permanent timestamp anchored in Bitcoin.

Your privacy is the point

Ordpool is the shield. We carry every external request for you so no third party sees who's stamping or verifying what.

Your file never leaves your browser

We compute the 32-byte SHA-256 fingerprint locally. Only the hash is ever transmitted. Never the contents, never the filename.

Calendars only see us

Stamp submissions and confirmation polls both route through our backend. The four public calendar operators receive ordpool's IP, not yours, on every request.

We don't log who you are

Our backend keeps no per-request user data: no IPs, no hashes, no timestamps in any logfile we control.

One honest caveat: Cloudflare sits in front of ordpool.space for DDoS protection and records standard CDN metadata (your IP, your country, the request path). If you don't trust them either, run a VPN on top. Then there's no path back to you.
